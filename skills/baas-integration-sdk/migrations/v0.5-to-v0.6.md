# v0.5 → v0.6 마이그레이션

동적 컬렉션(`collection`) **읽기 함수 선택 가이드 보강** 릴리스 — SDK 표면·서버 동작은 불변,
소비 방법의 문서 공백만 메움. (실 E2E에서 드러난 오류: `read:public` 컬렉션의 단건 상세를 인증용
`fetchRecord`로 조회 → 비로그인 401 → 반환 null 을 `.items` 접근해 크래시.)

## 무엇이 바뀌었나 (스킬 문서만)
- **인증 읽기(`fetchRecords`/`fetchRecord`)는 `read:public`이어도 항상 로그인 세션 필요** — 이 함수들은
  인증 데이터플레인 라우트라 컬렉션의 read 정책과 무관하다. 비로그인 조회는 공개 함수로만.
- **공개 단건 조회 = `BaasSDK.getPublicRecord(name, id)`** (top-level, SDK 표면). `useCollection` 훅엔
  래퍼가 없고, `fetchPublicRecords`는 레코드 `id` 필터가 안 되므로(`unknown_field`) 단건에 못 쓴다.
- **인증 읽기는 비로그인(401) 시 `BaasError` throw가 아니라 `null`을 resolve** → 반환값을
  `res?.items ?? []`/null 로 가드. (쓰기·기타 작업은 실패 시 `BaasError`(.message) throw — 상단 계약.
  에러를 토스트/모달/인라인 중 무엇으로 보여줄지는 **앱 UX 소관**이라 스킬이 규정하지 않는다.)
- SDK 코드/서버 access 모델/스키마 문법 변화 **없음**.

## 영향 / Tier (판정: features_used ∋ `collection`)
- **Tier 0**: `read:public` 컬렉션의 **단건 상세를 비로그인에 노출**하는 화면이 없으면 무영향.
- **Tier 2(국소 수정)**: 공개 컬렉션 단건 상세를 비로그인에 노출하는 앱만:
  - 단건 조회를 `fetchRecord` → `BaasSDK.getPublicRecord`(또는 목록 조회 후 클라이언트 `id` 매칭)로 교정.
  - 인증 읽기 결과를 직접 `.items` 접근하는 코드에 `?.items ?? []`/null 가드 추가.

## 적용 절차 (앱빌더/에이전트)
- `read:public` 컬렉션의 단건 상세 라우트를 찾아 공개 단건 조회로 교정한다.
- 인증 읽기(`fetchRecords`/`fetchRecord`) 반환값을 직접 `.items` 로 접근하는 지점에 null 가드를 추가한다.
