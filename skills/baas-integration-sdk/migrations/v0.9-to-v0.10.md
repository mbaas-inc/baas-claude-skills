# v0.9 → v0.10 마이그레이션

**구매약관 조회를 결제 공통 표면으로 일원화.** 결제(스토어·예약·커스텀)가 공유하는 약관 고지를
store 훅에 묶여 있던 것에서 떼어내 **`usePayment()`** 공통 훅으로 옮겼다. pre-1.0 정리라 **하위호환 별칭
없이 제거**했다(깨끗한 단일 정본).

## 무엇이 바뀌었나
- **신규**: `BaasSDK.usePayment()` 훅 — `{ fetchTerms, loading, error }`. 코어 `BaasSDK.getPurchaseTerms()`.
  - `fetchTerms()` = 표준 구매약관(통신판매중개 고지·결제진행 동의·환불정책·개인정보 제공 4조항) 조회.
    결제가 들어가는 모든 화면에서 결제 진입 전 `content` 표시 + 동의에 사용(반환 `{ title, content, version }`).
- **제거(BREAKING)**:
  - `BaasSDK.getStoreTerms()` → `BaasSDK.getPurchaseTerms()` (동일 엔드포인트, 이름만 정본화)
  - `useStore().fetchTerms()` → `usePayment().fetchTerms()` (약관은 store 전용이 아니라 결제 공통)
- **결제 실행 표면은 불변**: `useStore().beginWidgetCheckout` / `useReservation().beginWidgetCheckout` /
  `confirm` 등 그대로. 금액을 다루는 prepare/confirm 은 여전히 store/reservation 소유(안전상 서버 소유).
- **통신판매중개 고지 푸터 문구**를 `fetchTerms` 1번 조항 API 정본과 동일 문구로 정렬:
  "주식회사 엠바스는 통신판매중개자이며, 통신판매의 당사자가 아닙니다. 상품, 상품정보, 거래에 관한
  의무와 책임은 판매자에게 있습니다."
- **표면 문서**: `reference/sdk-surface.md` 에 `## 결제 (payment) — 공통 규약` 신설(위젯 방식 + ①구매약관
  동의 + ②통신판매중개 고지 푸터 + 커스텀 결제 원칙). store/예약 절은 이 공통 규약을 참조하도록 축약.

## 영향 / Tier (판정: features_used ∩ {payment, store, reservation})
- **Tier 0**: 결제(store·reservation·결제) 미사용 앱 → 교집합 없음 → 무영향.
- **Tier 2 (국소 수정)**: 구매약관 동의 UI를 `useStore().fetchTerms`(또는 `getStoreTerms`)로 구현한 앱:
  - `usePayment()` 훅을 추가하고 약관 조회를 `usePayment().fetchTerms()` 로 교체.
  - 통신판매중개 고지 푸터 문구를 위 API 정본과 동일하게 정렬.

## 적용 절차 (앱빌더/에이전트)
1. 약관을 쓰던 컴포넌트에 `const payment = BaasSDK.usePayment()` 추가.
2. `store.fetchTerms()` / `BaasSDK.getStoreTerms()` 호출을 `payment.fetchTerms()` / `BaasSDK.getPurchaseTerms()`
   로 교체. (반환 형태 `{ title, content, version }` 동일 — 렌더 코드 자체는 변경 불필요.)
3. 통신판매중개 고지 푸터 마지막 문장을 API 정본 문구로 정렬.
4. `npm run validate && npm run build` 확인 후 재배포. `baas-manifest.json` 의 `skill_version` → `"0.10.0"`,
   결제 사용 앱이면 `features_used` 에 `store`/`reservation`(+개념상 `payment`)이 실제로 들어있는지 확인.

### 참고: 코어 이름 변경은 기계적(선택적 codemod)
`BaasSDK.getStoreTerms` → `BaasSDK.getPurchaseTerms` 는 단순 식별자 치환이라 codemod 가능하나,
`useStore().fetchTerms` → `usePayment().fetchTerms` 는 훅 추가가 필요해 전체를 Tier 2(수동/LLM)로 둔다.
