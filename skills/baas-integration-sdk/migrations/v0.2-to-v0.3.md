# v0.2 → v0.3

## changed_features
- `collection` — 동적 컬렉션(사용자 정의 데이터 모델: 스키마 정의 + 레코드 CRUD)

## tier
- **N/A (추가형 신규 기능)** — 기존 배포 앱엔 마이그레이션 부담 없음.
  collection 은 v0.3 에서 **새로 추가**된 기능 그룹이므로, 기존 앱의 `features_used` 엔 collection 이 없다
  → 업데이트 판정 시 `features_used ∩ [collection] = ∅` → **건너뜀**(변경 대상 아님).
  collection 을 새로 쓰려는 앱은 마이그레이션이 아니라 **신규 생성** 경로다.

## 배경
- SDK 에 `useCollection` / `core/collection.ts` 프리미티브 추가(이슈 #608). 표면 사용법은 `reference/sdk-surface.md` "동적 컬렉션" 섹션.
- 접근 정책(`settings.access` — CRUD × `public/member/owner`)은 **서버가 강제 + CLI(`baas collection create --access-json`) 소유**다. SDK/스킬 표면 변경이 아니므로(이슈 #620) 이 노트의 대상 아님.
