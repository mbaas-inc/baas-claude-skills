# v0.6 → v0.7 마이그레이션

**파일 업로드(`storage`) 기능 신설** — presign 방식 업로드가 SDK 표면에 first-class 로 추가됐다.
**순수 추가(additive)** 라 기존 배포 앱은 무영향(아래 Tier 0).

## 무엇이 바뀌었나
- **신규 훅 `BaasSDK.useFileUpload()`** + 코어 `BaasSDK.uploadFile(file, { category })`.
  - `POST /upload/presign` 로 서명 URL 발급 → `presign_url` 로 S3 직접 PUT → `{ cdn_url, download_url, key, file_id? }` 반환.
  - 큰 바이너리가 CloudFront/Lambda 를 우회(서버 경유 업로드의 413/403·지연 해소).
- **표면 문서**: `reference/sdk-surface.md` "파일 업로드 (storage)" 절 추가. 동적 컬렉션 절과 교차 링크
  (이미지 필드는 `useFileUpload` 로 `cdn_url` 을 얻어 string(url) 필드에 저장).
- `category` 화이트리스트(기본 `images`): `images`(이미지 확장자·최대 10MB) | `store` | `reservation` |
  `board_import` | `board_attachment`. `board_attachment` 만 게시글 연결용 `file_id` 반환.
- SDK 서버 동작/기존 기능 표면 변화 **없음**.

## 영향 / Tier (판정: features_used ∋ `storage`)
- **Tier 0 (전 기존 앱)**: `storage` 는 신규 기능이라 기존 `baas-manifest.json` 의 `features_used` 에
  들어있지 않다 → 교집합이 항상 비어 **갱신 대상 아님**. 별도 조치 불필요.
- 새 앱에서 이미지/파일 업로드가 필요하면 `useFileUpload` 를 사용하고 `manifest.features_used` 에 `storage` 를 기록한다.

## 적용 절차
- 기존 앱: 없음(추가 기능).
- 신규 앱: 파일 입력 UI 에서 `useFileUpload().upload(file)` → 반환 `cdn_url` 을 콘텐츠(컬렉션 이미지 필드·`<img src>`)에 저장.
